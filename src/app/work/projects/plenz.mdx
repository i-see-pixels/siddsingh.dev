## Overview

plenz is an open-source extension and product system for improving AI prompts while you type. The idea is simple: make prompt quality visible in the moment instead of expecting people to rewrite prompts after the fact.

## What I built

- A real-time prompt refinement experience that works across major AI chat platforms.
- A provider-agnostic architecture so different model backends can power the same product experience.
- A privacy-first setup built around BYOK, local-first storage, and no central prompt-processing backend.
- A monorepo structure spanning the browser extension, web app, VS Code extension, shared core logic, and provider adapters.

## Why it matters

plenz reflects the kind of products I like building: fast feedback loops, clear user value, and a technical architecture that can grow without becoming hard to reason about. It also sits right at the intersection of product engineering and AI tooling, which is where a lot of my current curiosity lives.

## Stack

- Next.js for the web app
- Vite and Preact for the browser extension
- TypeScript across the monorepo
- Shared packages for core logic, UI, and provider integrations

## Outcome

The project gave me a strong sandbox for thinking through product UX, extension constraints, AI-provider abstractions, and how to ship one idea coherently across multiple surfaces.

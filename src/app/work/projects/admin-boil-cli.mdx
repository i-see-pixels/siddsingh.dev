## Overview

admin-boil CLI is a command-line tool for dropping the admin-boil dashboard into a project with minimal setup. The focus was on reducing the friction that usually comes with wiring together dashboards, dependencies, and framework-specific configuration.

## What I built

- A CLI that can initialize a new project or configure an existing codebase.
- Support for modern frameworks including Next.js, Vite, Astro, and other frontend setups.
- A smoother onboarding path so developers can go from install to a usable admin dashboard quickly.
- A foundation for packaging a productized developer experience instead of shipping only UI files.

## Why it matters

This project leans into a pattern I enjoy a lot: taking repetitive setup work and turning it into a simpler developer workflow. Instead of asking people to copy boilerplate by hand, the CLI handles the mechanical steps and lets them start from a stronger baseline.

## Stack

- TypeScript
- Node.js CLI tooling
- Framework-aware project setup logic
- A companion product site for distribution and onboarding

## Outcome

admin-boil CLI helped clarify how I think about developer experience as a product. Good tooling is not just about code generation. It is about removing hesitation, shortening time-to-value, and making the first five minutes feel obvious.
